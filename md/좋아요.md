# Consumer–Provider 소셜 API (Favorite / Like / Recommend)

앱에서 **1️⃣ Favorite Provider**, **2️⃣ I Like Him**, **3️⃣ Recommend Provider** 기능을 사용하기 위한 API 및 연동 가이드입니다.

---

## 전제

- **Base URL**: `https://<your-api-host>`
- **인증**: 모든 API는 **JWT Bearer** 필요. `Authorization: Bearer <access_token>`
- **providerId**: 모든 요청/응답의 `providerId`는 **`providers.id`** (UUID). 서버 내부에서 provider의 `user_id`로 변환해 저장합니다.

---

## 1️⃣ Favorite Provider (찜)
### — 배치 위치
**Provider 프로필 카드** 어디서든 가능하게:

- 견적 비교 화면 (Bid Comparison Screen) → 각 Provider 카드 우상단 ♡ 버튼
- Provider 상세 프로필 화면 → 상단 헤더 영역 ♡ 버튼
- 서비스 완료 후 리뷰 화면 → "Save to Favorites" 옵션

**소비자 My Page → "My Favorite Providers"** 섹션에서 모아보기

| 메서드 | 경로 | 설명 |
|--------|------|------|
| POST | `/api/v1/consumer-provider/favorites` | 프로바이더를 찜 목록에 추가 |
| DELETE | `/api/v1/consumer-provider/favorites/:providerId` | 찜 해제 |
| GET | `/api/v1/consumer-provider/favorites` | 내 찜 목록 조회 |
| GET | `/api/v1/consumer-provider/favorites/check/:providerId` | 해당 프로바이더 찜 여부 확인 |

### POST `/api/v1/consumer-provider/favorites`

**Body (JSON)**

```json
{
  "providerId": "123e4567-e89b-12d3-a456-426614174000"
}
```

**응답 (201)**

```json
{
  "id": "uuid-of-favorite-row",
  "providerId": "123e4567-e89b-12d3-a456-426614174000",
  "createdAt": "2025-03-07T12:00:00.000Z"
}
```

### DELETE `/api/v1/consumer-provider/favorites/:providerId`

**응답 (200)**

```json
{
  "deleted": true
}
```

### GET `/api/v1/consumer-provider/favorites`

**응답 (200)**

```json
[
  {
    "id": "uuid",
    "providerId": "provider-uuid",
    "createdAt": "2025-03-07T12:00:00.000Z"
  }
]
```

### GET `/api/v1/consumer-provider/favorites/check/:providerId`

**응답 (200)**

```json
{
  "isFavorite": true
}
```

## 2️⃣ I Like Him (좋아요)

— 배치 위치

서비스 **완료 후** 리뷰/평가 화면에서만 표시:

- 별점 + 리뷰 작성 화면 하단에 👍 "I'd work with this provider again" 토글
- Provider 공개 프로필에 Like 수 표시 (Trust Score 보조 지표)


| 메서드 | 경로 | 설명 |
|--------|------|------|
| POST | `/api/v1/consumer-provider/reactions/like` | 좋아요 추가 |
| DELETE | `/api/v1/consumer-provider/reactions/like/:providerId` | 좋아요 취소 |
| GET | `/api/v1/consumer-provider/reactions/like` | 내가 좋아요한 프로바이더 목록 |

### POST `/api/v1/consumer-provider/reactions/like`

**Body (JSON)**

```json
{
  "providerId": "123e4567-e89b-12d3-a456-426614174000"
}
```

**응답 (201)**

```json
{
  "id": "uuid",
  "providerId": "123e4567-e89b-12d3-a456-426614174000",
  "reaction": "like",
  "createdAt": "2025-03-07T12:00:00.000Z"
}
```

### DELETE `/api/v1/consumer-provider/reactions/like/:providerId`

**응답 (200)**  
`{ "deleted": true }`

### GET `/api/v1/consumer-provider/reactions/like`

**응답 (200)**  
배열. 각 항목: `{ id, providerId, reaction: "like", createdAt }`

## 3️⃣ Recommend Provider (추천)
— 배치 위치

서비스 **완료 후** 또는 Provider 프로필 화면에서:

- 리뷰 완료 후 "Share this Provider" 버튼
- Provider 프로필 화면 → Share 아이콘 → 앱 내 추천 or 외부 공유
- "추천 코드" 방식으로 친구에게 링크 공유 → Reward Credits 연계 가능

| 메서드 | 경로 | 설명 |
|--------|------|------|
| POST | `/api/v1/consumer-provider/reactions/recommend` | 추천 추가 (한마디·완료 계약 선택) |
| DELETE | `/api/v1/consumer-provider/reactions/recommend/:providerId` | 추천 취소 |
| GET | `/api/v1/consumer-provider/reactions/recommend` | 내가 추천한 프로바이더 목록 |

### POST `/api/v1/consumer-provider/reactions/recommend`

**Body (JSON)**

```json
{
  "providerId": "123e4567-e89b-12d3-a456-426614174000",
  "note": "Very professional, on time!",
  "bookingId": "optional-booking-uuid-for-verification",
  "isPublic": true
}
```

- **note**: (선택) 추천 한마디. 최대 500자.
- **bookingId**: (선택) 완료된 예약 ID. 있으면 “완료 계약 기반 검증”으로 저장.
- **isPublic**: (선택) 공개 여부. 기본 `true`.

**응답 (201)**

```json
{
  "id": "uuid",
  "providerId": "123e4567-e89b-12d3-a456-426614174000",
  "reaction": "recommend",
  "note": "Very professional, on time!",
  "createdAt": "2025-03-07T12:00:00.000Z"
}
```

### DELETE `/api/v1/consumer-provider/reactions/recommend/:providerId`

**응답 (200)**  
`{ "deleted": true }`

### GET `/api/v1/consumer-provider/reactions/recommend`

**응답 (200)**  
배열. 각 항목: `{ id, providerId, reaction: "recommend", note, createdAt }`

## 프로바이더별 집계 및 내 상태

### GET `/api/v1/consumer-provider/stats/:providerId`

프로바이더의 **소셜 집계** (materialized view `provider_social_stats` 기반).

**응답 (200)**

```json
{
  "providerId": "123e4567-e89b-12d3-a456-426614174000",
  "favoriteCount": 10,
  "likeCount": 5,
  "recommendCount": 3
}
```

- **favoriteCount**: 이 프로바이더를 찜한 소비자 수.
- **likeCount**: 이 프로바이더에게 좋아요한 소비자 수.
- **recommendCount**: 이 프로바이더를 추천한 소비자 수.

> 집계는 MV를 읽습니다. 최신 반영을 위해 MV 새로고침이 필요하면 DB에서  
> `REFRESH MATERIALIZED VIEW provider_social_stats;` 실행.

### GET `/api/v1/consumer-provider/me/state/:providerId`

**현재 로그인한 소비자**가 해당 프로바이더에 대해  
찜 / 좋아요 / 추천을 했는지 여부.

**응답 (200)**

```json
{
  "isFavorite": true,
  "isLike": false,
  "isRecommend": true
}
```

## 앱 연동 시 참고

1. **providerId**  
   - 요청·응답 모두 **`providers.id`** (UUID). 프로바이더 상세/목록에서 쓰는 ID와 동일하게 사용하면 됩니다.

2. **에러**  
   - **404**: 해당 `providerId`에 대한 프로바이더가 없음.  
   - **401**: 토큰 없음/만료. 재로그인 후 재시도.

3. **Swagger**  
   - `GET /api` (또는 Swagger UI 경로)에서 **consumer-provider** 태그로 위 API를 확인할 수 있습니다.

4. **DB 테이블**  
   - `consumer_provider_favorites`: 찜  
   - `consumer_provider_reactions`: 좋아요/추천 (reaction = `like` | `recommend`)  
   - `provider_social_stats`: 프로바이더별 집계 (materialized view)

5. **중복**  
   - 찜: (consumer_id, provider_id) 유니크.  
   - 반응: (consumer_id, provider_id, reaction) 유니크.  
   동일 요청 재전송 시 기존 행이 있으면 그대로 반환/유지됩니다.
